This directory contains binaries of all libraries used by
Proximity. They are located here for convenience, instead of making
users install each package. Note that some jars depend on
others. Names and versions:


* colt.jar

purpose: scientific and technical computing libraries (used in adjacency matrix,
  spectral clustering). also used by jung
version: 1.2.0
url: http://dsd.lbl.gov/~hoschek/colt/


* commons-collections-3.1.jar
purpose: extends the Java Collections Framework. used by jung
version: 3.1
url: http://jakarta.apache.org/commons/collections/


* jdom.jar

purpose: XML package
version: jdom-1.0
url: http://www.jdom.org/


* jung-1.7.6.jar

purpose: graph data structure and visualization package
version: 1.7.6
url: http://jung.sourceforge.net/


* junit.jar

purpose: unit testing package
version: 3.8.1
url: http://www.junit.org/index.htm


* jython.jar

purpose: Python implementation in Java
version: 2.1
url: http://www.jython.org/


* log4j-1.2.14.jar

purpose: logging package
version: log4j 1.2.14
url: http://jakarta.apache.org/log4j/docs/


* mantissa-7.0.jar

purpose: Mathematical Library. Used for NSIs.
version: 7.0
url: http://www.spaceroots.org


* piccolox.jar, piccolo.jar

purpose: 2D graphics package
version: CVS version as of 2004-11-18 18 09:31
url: http://www.cs.umd.edu/hcil/jazz/


* spin.jar

purpose: Swing threading helper library
version: 1.5
url: http://spin.sourceforge.net/


* xercesImpl.jar

purpose: Higher-quality XML parser than the one available in JDK 1.4.2. also used by jung
version: 2.6.0
url: http://www.apache.org/dyn/closer.cgi/xml/xerces-j/
